# Шаблон профилирования должности

Этот документ описывает текущую MVP-анкету, реализованную во фронтенде в
`apps/web/src/data/questionnaire.ts`.

MVP-интерфейс является управляемым заполнением утверждённого шаблона, а не
ИИ-диалогом. Система не задаёт дополнительные уточняющие вопросы, не
переформулирует ответы и не оценивает полноту автоматически.

## Общие UX-правила MVP

- Раздел 1 показывается одной страницей, потому что содержит связанную общую информацию.
- Разделы 2-9 показываются по одному логическому блоку за раз.
- Справа отображается статическая справка по текущему вопросу или активному полю.
- Для карточек используется добавление одного элемента за раз, без больших пустых таблиц.
- Отдельного варианта для неопределённого ответа нет: пользователь выбирает применимый вариант или "Не применимо", если это предусмотрено вопросом.
- Если доступен вариант "Не применимо" или "Не требуется", он должен исключать одновременный выбор других вариантов в multi-choice.
- В требованиях к должности пользователь заполняет требования к роли, а не свои личные данные.
